SECURE PROJECT WORKSPACE SETUP REPORT


1. Command: mkdir -p secure_workspace/project_docs && touch secure_workspace/project_docs/data.txt
Explanation: I used "mkdir -p" to create a parent and child directory structure for the workspace, and "touch" to create a blank project file inside it. This sets up the basic hierarchy required for the team.


2. Command: umask
Output: 0022
Explanation: I ran the "umask" command to view the default permission mask applied to newly created files and directories. My system returned "0022", meaning new directories default to 755 and new files default to 644.


3. Command: ls -l secure_workspace/project_docs/data.txt (Before Modification)
Output: -rw-r--r-- 1 swapnil swapnil 0 Jun 23 20:21 secure_workspace/project_docs/data.txt
Explanation: I used "ls -l" to check the default permissions of the newly created file. As expected by the umask, the file was created with read and write permissions for the owner, and read-only for others.


4. Command: chmod 750 secure_workspace && chmod 640 secure_workspace/project_docs/data.txt
Explanation: I used "chmod 750" on the directory so only the owner and group can read/execute it, blocking public access. I used "chmod 640" on the file so the owner can read/write, the group can read, and no one else has access.


5. Command: ls -l secure_workspace/project_docs/data.txt (After Modification)
Output: -rw-r----- 1 swapnil swapnil 0 Jun 23 20:21 secure_workspace/project_docs/data.txt
Explanation: I ran "ls -l" again to verify the permission changes were successfully applied. 


Explanation of Protection:
By modifying the permissions to 750 (directories) and 640 (files), I have ensured that only the file owner can modify the data, while project team members (the group) can read the data. All other users on the system have zero access, keeping the project secure.
