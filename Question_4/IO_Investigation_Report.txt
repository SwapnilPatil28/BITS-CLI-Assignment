3. Command: ls -l /nonexistent_dir > output.txt 2> error.txt
Explanation: I purposely ran a command that would fail, using standard output redirection (">") to send successful results to "output.txt" and standard error redirection ("2>") to send error messages to "error.txt".


4. Command: cat output.txt && cat error.txt
Output: swapnil@Swapnil-Patil:~$ cat output.txt
        swapnil@Swapnil-Patil:~$ cat error.txt

Explanation: I checked the contents of both files. "output.txt" was empty because the command did not succeed. "error.txt" contained the "No such file or directory" message, proving that Linux successfully separated the standard error stream from the standard output stream.


Explanation of Linux File I/O:
Linux treats everything as a file and manages I/O using File Descriptors (FDs). By default, FD 0 is Standard Input (stdin), FD 1 is Standard Output (stdout), and FD 2 is Standard Error (stderr). This architecture allows engineers to easily redirect, log, and isolate different types of process outputs.
